# 战斗 UI 布局 v0

## 目标

当前项目视口为 `640×360`，棋盘本身仍是 20×20、每格 32 像素。v0 布局的目标不是最终美术，而是先解决 MVP 中“信息面板遮挡棋盘、文字散落、操作入口到处漂浮”的问题。

原则：
- 棋盘是主画面，常驻 UI 只能占边缘。
- 重要操作集中到底部，不再跟随单位弹出长菜单。
- 日志、伤害预览和未来单位详情不能同时铺开，统一使用右侧抽屉。
- 棋盘中央只显示临时反馈：范围、高亮、伤害数字、状态短标签和预警格。

## 屏幕分区

### 顶部状态条

顶部只放低矮信息：
- 左侧：CTB 行动轴，默认展示未来行动顺序；玩家可切换到跑条视图。
- 中部：同步率小 HUD，展示同步率、自然回复状态、训练师形态、后备和冷却摘要。
- 右侧：全体敌方范围按钮、日志抽屉按钮。

顶部状态条不承担长文本说明；长说明交给底部提示线或右侧抽屉。

### 底部指令栏

当前单位行动时，行动菜单固定在屏幕底部，作为一条指令栏：
- 基础动作：移动、技能、待机/结束。
- 训练师动作：同步率顶层入口，再展开指令、提取、召唤。
- 指令栏可以换行和滚动，但不能跟随单位位置漂浮。

选择移动、技能目标、指令卡目标、召唤格或回收目标时，底部指令栏隐藏，让玩家专注于棋盘选择。

### 右侧抽屉

右侧抽屉是可遮挡区域，但默认关闭：
- 点击“日志”才展开行动日志。
- 技能预览时临时占用同一区域，并自动隐藏日志。
- 未来如果加入单位详情，也应放进同一抽屉中，以标签或模式切换呈现。

右侧抽屉与底部指令栏不能同时无限扩张；内容过多时使用滚动。

### 底部提示线

提示文本放在底部指令栏上方，用短句解释当前状态、目标选择和非法操作原因。它不承载完整攻略，也不写“适合做什么”这类策略答案。

### 中央弹窗

战前准备、第一场战斗说明和战斗结算仍使用中央弹窗，因为这些阶段 CTB 不推进或战斗已经结束，不属于常驻遮挡。

## 当前取舍

v0 仍是叠加在棋盘上的 UI，没有真正改变相机和棋盘可视区域。后续如果要完全避免遮挡，需要做以下之一：
- 提高设计分辨率。
- 加 Camera2D 并给棋盘可滚动安全区。
- 做“战场视口 + UI 外框”的布局，让棋盘不绘制在 HUD 底下。

MVP 阶段先验证信息分区和操作节奏，再决定是否进入正式战斗 UI 重构。
