# 单位与技能池设计稿

## 目标
这份文档记录 MVP 样板战当前的可用精灵、提取形态和技能定位。它的作用不是给玩家写攻略，而是给后续调参、扩展精灵和设计第二关时提供一个一致基线。

当前原则：
- 每只可用精灵至少有一个清楚定位，不只是换属性的伤害按钮。
- 大部分技能仍是标准行动；只有少量快招/重招显示释放者自身行动提前或推后。
- 控制技能先只做轻量移动压制，不引入复杂冰冻、眩晕或跳过行动。
- 训练师提取只获得对应精灵的一个技能；不同精灵可以提供攻击、治疗或控制等不同提取价值。

## 移动压制
移动压制是当前 MVP 的第一种技能型减益。

规则：
- 技能附加效果写入 `SkillData.effects`。当前移动压制使用 `ADD_STATUS + MOVE_PENALTY`，行动条压制使用 `AP_DELTA`。
- 移动压制使目标下次行动的可移动距离减少对应数值。
- 该状态在目标行动结束后清除，即使目标没有移动也会消失。
- 状态短标签为“缚”，归类为减益，持续类型为“下次行动有效”。

当前使用者：
- 藤藤兽 / 训练师草形态：缠绕，移动-1，快招。
- 冰羽兽：霜缚，移动-2，目标行动条-20%。

## 防御变化
防御变化是当前 MVP 第一种通用 `StatusInstance` 数值状态。

规则：
- 技能附加效果使用 `ADD_STATUS + DEFENSE_MOD`。
- 目标获得“防-数字”状态，当前持续类型为“下次受伤有效”。
- 目标下次受到伤害前，`StatusResolver` 会把防御变化计入有效防御；结算后消耗该状态。
- 该状态使用 `REPLACE_STRONGER` 叠加规则，重复施加时保留更强的一次。

当前使用者：
- 冰羽兽 / 训练师冰形态：冰针，防御-3。

## 后续注意
- 如果继续加精灵，优先补“定位空缺”，例如纯前排、反应型、防御型、地形型，而不是只补新属性。
- 如果某个技能只有数值更高、没有位置/节奏/属性上的区别，应该先检查它是否真的需要存在。
- 技能说明应描述机制事实，例如“范围 1”“移动-1”“自身下次行动提前 20%”“目标行动条-20%”，避免直接告诉玩家“适合怎么打”。
